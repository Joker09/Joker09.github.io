CSS盒模型

这里不多说，直接引用MDN上的介绍吧！——盒模型-MDN

文档中的每个元素被描绘为矩形盒子。渲染引擎的目的就是判定大小，属性——比如它的颜色、背景、边框方面——及这些盒子的位置。

在CSS中，这些矩形盒子用 标准盒模型 来描述。这个模型描述了一个元素所占用的空间。每一个盒子有四条边界：外边距边界margin edge, 边框边界border edge, 内边距边界padding edge 与 内容边界content edge。

绝对定位和相对定位

先说比较容易理解的相对定位。如果对一个元素进行相对定位（positon:relative），它在原本的位置上不会移动（整个页面布局还是原来的样子）。然后，通过设置垂直或水平位置（设置left和top的值），让这个元素“相对于”它在标准文档流中的位置进行移动。看下面的例子：

#box2 {
    postion:relative;
    left:30px;
    top:20px;
}

注意：在使用CSS相对定位时，无论是否进行移动，元素仍然占据原来的空间。即原来的空间不会被关闭。

再来看绝对定位。如果对一个元素进行绝对定位（positon:absolute），那么它会从标准文档流中脱离，原来所占据的空间会被关闭，就好像该元素不存在一样，由它所导致的空白文本节点也会消失。首先，如果没有给这个绝对定位的元素添加left和top值的话，那么元素的left和top值为auto，并非为0。所以从视觉上来看，它会浮在它原来位置的上方。设置了left和top之后，它会相对于最近的已定位的（position为relative/absolute/fixed）的祖先元素来定位，即若设置left和top均为0，则它与已定位的祖先元素的内边距边界的左上角对齐。如果没有已定位的祖先元素，则相对于它最初的包含块（chrome和IE均指html）定位。看下面例子：

#box2 {
    postion:absolute;
    left:30px;
    top:20px;
}

注意：绝对定位后，不管元素之前的display是block还是inline都会变成inline-block。
