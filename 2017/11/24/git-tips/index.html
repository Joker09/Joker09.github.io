常用git命令备忘

2017-11-24
kkk
杂
git

最近公司全部上git了，这里记录一下经常用到的命令备忘。

远程已有仓库

首先从远程仓库下载已有的代码

git clone http://path/to/repository.git
# 或者
git clone username@paht/to/repository.git

进入下载好的仓库文件夹，使用git remote命令，可以看到本地自动为你添加了相应的远程仓库，并且它的名字叫origin。

在本地修改代码后commit后，初次向远程仓库提交数据需要使用下面这行命令：

git push -u origin master:master

解释一下命令的意思：

  • -u 完整的写法是-set-upstream，也就是将本地仓库的分支与远程仓库的分支建立追踪联系

  • origin 表示的是远程仓库的名字

  • master:master 前一个master表示的是本地的master分支，后一个master表示的是远程仓库的master分支

所以这行命令翻译成大白话就是：

hey，git老兄，请帮我将本地仓库的master分支与一个名叫origin远程仓库的master分支建立追踪关系吧，然后再把本地的一些新数据更新到远程上去。

Tips:

  • -u 只需要在初次提交数据时添加，以后可直接使用git push origin master:master

  • 如果本地仓库分支与要提交到的远程仓库分支同名，则可以简写为git push origin master

  • git push 有一些默认行为，这也就是为什么可以再将上面的命令直接简写为git push 便可以向远程提交数据的原因。关于它的行为，可以参考这里

本地已有仓库

本地已有代码仓库的话，需要在远程建立一个新的空仓库，仓库建好之后，一般会提示这么一段话：

从命令行推送已经创建的仓库

  git remote add origin root@git.heifun.cn:kkk/test2.git
  git push -u origin master

这两句话的意思就是：

添加一个地址为root@git.heifun.cn:kkk/test2.git远程仓库，并把它命名为origin，然后将这个名叫origin的远程仓库的master分支与本地仓库的master分支建立追踪关系，并将本地master分支上的数据推送到远程master分支上。

直接照做即可。
